# TradingClaw 命令行详细设计

## 1. 设计目标与范围

### 1.1 设计目标
- 提供统一命令入口，覆盖 MVP 核心查询、交易、策略能力。
- 同时满足人工交互与自动化脚本调用。
- 保持输出契约稳定，失败可判定，敏感信息不泄露。

### 1.2 范围说明
- 覆盖统一入口、全局参数、输出规范、配置与会话缓存。
- 覆盖用户、交易所会话、行情资产、交易、策略、资讯命令组。
- 不包含图形化流程编排、交易所登录授权实现、GUI 安装器。

## 2. CLI 总体架构

### 2.1 分层架构
- 命令层：命令注册、参数解析、帮助与示例输出。
- 应用层：用例编排、参数归一、前置校验、确认机制。
- 适配层：HTTP 客户端、输出渲染器、会话与配置存储适配器。
- 基础层：本地文件系统、日志、终端能力检测。

### 2.2 模块划分
- core：入口、全局选项、错误处理、退出码映射。
- output：JSON/表格渲染、分页器、格式化器。
- config：配置读写、合法键校验、目录初始化。
- session：用户会话缓存、交易所会话缓存、默认会话管理。
- commands.user：登录、资料、额度积分、登出。
- commands.exchange：交易所会话查询、设置、删除。
- commands.market：资产搜索、行情、持仓、历史查询。
- commands.trade：证券下单、撤单、扩展交易命令。
- commands.strategy：策略创建、列表、详情、停启、修改。
- commands.news：资讯查询。

### 2.3 目录建议
- cli/main.py：统一入口。
- cli/core：全局能力。
- cli/commands：按业务域拆分命令组。
- cli/services：后端调用封装。
- cli/storage：本地配置与会话存储。
- cli/output：输出与分页能力。

## 3. 命令体系设计

### 3.1 顶层命令结构
- tc user ...
- tc exchange ...
- tc market ...
- tc trade ...
- tc strategy ...
- tc news ...
- tc config ...

### 3.2 全局参数
- --base-url：服务地址。
- --output：json 或 table。
- --pretty：JSON 美化输出开关。
- --session-id：显式用户会话覆盖。
- --exchange-session-id：显式交易所会话覆盖。
- --page：分页页码或游标。
- --limit：返回数量限制。
- --no-pager：关闭分页输出。

### 3.3 参数覆盖规则
- 命令级参数优先级高于全局参数。
- 显式参数优先级高于本地默认配置。
- 同名参数冲突按就近原则生效并输出调试提示。

## 4. 输出与退出码设计

### 4.1 统一输出契约
- status：success 或 error。
- message：可读结果描述。
- data：业务数据载荷。
- error_code：失败时为可检索错误码。
- request_id：单次调用链路标识。
- timestamp：结果生成时间戳。

### 4.2 表格输出规范
- 表头固定字段顺序，避免脚本解析混乱。
- 列表过长时支持分页与数量裁剪。
- 空数据输出明确空态提示，不输出异常栈。

### 4.3 退出码规范
- 0：执行成功。
- 2：参数错误。
- 3：认证或会话错误。
- 4：权限与主体一致性错误。
- 5：业务规则错误。
- 6：外部依赖错误。
- 7：系统内部错误。

## 5. 本地配置与会话缓存设计

### 5.1 本地目录
- 首次执行自动创建用户目录下专用隐藏目录。
- 配置与会话分文件存储，避免相互污染。
- 目录与文件权限仅当前用户可读写。

### 5.2 配置模型
- base_url、output、pretty、default_exchange_session_id。
- 数字资产相关密钥配置项采用加密存储或系统安全存储适配。
- 非法键写入立即失败并返回可读原因。

### 5.3 会话缓存模型
- user_session：access_token、session_id、expire_at。
- exchange_sessions：会话列表与默认会话标识。
- 读取过期会话时阻断业务并提示重新登录。
- 登出后必须清理可复用凭证。

## 6. 各业务域命令详细设计

### 6.1 用户命令组（CLI-004）
- login：用户名密码登录并缓存会话。
- profile：读取用户资料。
- quota：读取策略额度。
- points：读取积分总量、占用与可用值。
- logout：清理本地会话并调用后端失效。

### 6.2 交易所会话命令组（CLI-005）
- list：查询当前用户可用交易所会话。
- get-default：读取默认会话。
- set-default：设置默认会话。
- delete：删除交易所会话并同步默认值。

### 6.3 行情与资产命令组（CLI-006/011）
- asset search：关键词资产搜索。
- stock quote/level/history：股票实时、五档、历史。
- fund quote/history：基金实时净值、历史净值。
- portfolio stock/fund：证券与基金持仓查询。
- ma 命令与数字资产行情命令在 MVP 返回能力未开放。

### 6.4 交易命令组（CLI-007/012/013）
- trade stock limit：股票限价下单。
- trade stock market：股票市价下单，含保护价前置校验。
- trade fund order：基金申购与赎回。
- trade stock cancel：股票单笔撤单。
- V1 扩展：批量撤单、流水查询、数字资产订单创建修改撤销查询。

### 6.5 策略命令组（CLI-008）
- strategy list/detail：列表与详情查询。
- strategy create：基础策略创建。
- strategy stop/restart/update：生命周期操作。
- 复杂参数支持 --json 与 --json-file 两种输入方式。
- JSON 解析失败必须阻断并给出可执行示例。

### 6.6 资讯命令组（CLI-009）
- news list：按数量和游标拉取资讯。
- 支持 JSON 与表格双格式输出。
- 多源聚合排序由后端结果驱动，CLI 不重排业务时间线。

## 7. 前置校验与确认机制

### 7.1 前置校验
- 参数完整性校验。
- 枚举值合法性校验。
- 市价保护价区间校验。
- 会话可用性校验。

### 7.2 高风险确认
- 交互环境默认弹出确认提示。
- 非交互环境必须要求 --yes 或等价显式确认参数。
- 缺失确认参数时返回非零退出码与示例命令。

## 8. 错误处理与可观测性

### 8.1 错误分类处理
- 参数错误：直接终止并输出参数纠正建议。
- 认证错误：提示登录或会话修复路径。
- 业务错误：原样展示业务失败原因与错误码。
- 系统错误：输出 request_id 供排障追踪。

### 8.2 日志策略
- 默认不打印敏感字段。
- 调试模式可输出请求耗时与重试信息，不输出明文凭证。
- 关键写操作记录最小化审计日志。

## 9. 非功能设计

### 9.1 稳定性
- 所有命令执行入口统一异常捕获。
- 后端错误不得导致无信息崩溃退出。
- 网络异常、参数异常、认证异常给出可定位反馈。

### 9.2 安全性
- 敏感字段全量脱敏。
- 令牌与密钥最小化存储。
- 会话过期自动阻断，不尝试静默绕过。

### 9.3 自动化友好
- JSON 契约保持稳定。
- 失败必须返回非零退出码。
- 支持环境参数化切换与批处理执行。

## 10. 发布与版本策略

### 10.1 MVP 范围
- CLI-001~CLI-008 全量交付。
- CLI-009~CLI-013 返回能力未开放提示与版本信息。

### 10.2 V1 范围
- 开放资讯、均线与数字资产行情、数字资产订单、证券订单扩展。
- 与后端版本矩阵严格同步，不超前承诺。

### 10.3 打包分发
- 提供 Python 标准打包产物。
- 提供容器化构建产物。
- 在 CI 中自动产出可安装包并执行冒烟验证。

## 11. 验收设计

### 11.1 功能验收
- 覆盖登录、会话、行情、交易、策略、配置核心链路。
- 覆盖 JSON 与表格双输出一致性。
- 覆盖失败退出码与错误消息可读性。

### 11.2 安全验收
- 脱敏字段覆盖率 100%。
- 登出后本地凭证不可复用。
- 非交互高风险命令无确认参数必须阻断。

### 11.3 回归验收
- 版本守卫与能力未开放口径稳定。
- 全局参数覆盖规则稳定。
- 新增命令组不影响既有命令行为与输出结构。
