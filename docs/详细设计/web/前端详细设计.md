# TradingClaw 前端详细设计

## 1. 设计目标与范围

### 1.1 设计目标
- 在前端需求文档约束下，形成可直接落地的实现设计。
- 保证 MVP 可交付，V1/V2 可平滑扩展，不破坏既有消费契约。
- 保证页面层配置驱动、文案统一供给、会话与权限链路一致。

### 1.2 范围说明
- 覆盖应用壳、导航、认证、账户、策略、文档中心、智能分析、券商辅助登录。
- 覆盖前端服务层：配置服务、文档服务、业务聚合服务。
- 不包含视觉稿细节、具体组件像素级规范与后端内部实现。

## 2. 架构设计

### 2.1 分层架构
- 表现层：页面、布局、业务组件、全局反馈组件。
- 应用层：路由、会话守卫、状态编排、主题编排、页面级用例。
- 前端服务层：配置服务、文档服务、业务聚合服务、缓存管理。
- 基础设施层：HTTP 客户端、错误映射、埋点、存储适配器。

### 2.2 模块划分
- shell 模块：应用壳、导航、主题、响应式容器。
- auth 模块：注册、登录、登出、会话恢复。
- account 模块：交易账户管理、模型设置读取与写入入口。
- strategy-intraday 模块：场内策略列表、创建、启停、更新、下线。
- strategy-offshore 模块：场外策略列表、创建编辑、启停、下线。
- strategy-history 模块：已完成策略查看、排序、收益表达。
- analysis 模块：智能分析对话、上下文增强、模型设置。
- broker-assist 模块：券商登录辅助、验证码流程、回跳。
- docs-center 模块：文档目录、正文阅读、移动端切换。

### 2.3 目录建议
- src/app：入口、路由、全局 Provider。
- src/layouts：应用壳与导航容器。
- src/features：按业务域拆分模块。
- src/services：配置服务、文档服务、业务聚合服务。
- src/stores：会话、配置、主题、页面状态。
- src/shared：通用组件、工具、常量、类型。

## 3. 路由与权限设计

### 3.1 路由分组
- 公共路由：登录、注册、文档中心（可配置为公开）、能力未开放页。
- 受限路由：账户设置、策略管理、智能分析、券商辅助登录。
- 回跳路由：券商辅助登录成功后返回触发来源页面。

### 3.2 路由守卫
- 未登录访问受限路由时重定向到登录页并记录来源地址。
- 登录成功后优先跳转来源地址，否则跳转默认业务页。
- 会话失效统一拦截，清理会话并触发全局重新登录引导。

### 3.3 能力版本守卫
- 页面进入前读取能力开关与版本矩阵。
- 未开放能力显示统一占位态，包含版本信息与替代路径。
- 未开放能力阻断提交动作，不以异常弹窗替代。

## 4. 前端服务层详细设计

### 4.1 配置服务
- 输入：页面标识、配置版本、语言上下文。
- 输出：全局配置、页面配置、文案字典、降级文案。
- 缓存：内存缓存为主，持久缓存为辅，支持 TTL 与手动失效。
- 失败策略：读取失败时启用兜底文案并标记来源为 fallback。

### 4.2 文档服务
- MVP 数据源：前端静态资源或静态配置。
- 输出能力：目录列表、文档正文、结构化章节信息。
- 状态管理：加载中、加载失败、空数据三态统一。
- 移动端：目录与阅读区互斥切换。

### 4.3 前端业务聚合服务
- 统一封装页面调用接口，注入会话参数与请求追踪标识。
- 统一错误映射，输出可读 message 与标准 error_code。
- 统一返回结构，页面层只消费稳定契约字段。
- 统一管理关键请求重试与幂等提示策略。

## 5. 状态管理设计

### 5.1 全局状态域
- 会话状态：access token、用户信息、会话标识、交易所会话标识。
- 主题状态：深浅主题、跟随系统、持久化配置。
- 配置状态：全局配置、页面配置、文案字典、版本标识。
- 运行时状态：全局 loading、全局通知、网络在线状态。

### 5.2 页面状态域
- 列表页状态：筛选条件、分页、排序、骨架态、空态、错误态。
- 表单状态：字段值、校验状态、提交状态、失败原因。
- 对话状态：消息列表、发送状态、上下文对象、模型参数。

### 5.3 状态一致性规则
- 会话状态变化后，路由权限和页面可操作状态必须同步更新。
- 列表类操作成功后主动刷新受影响列表或局部合并更新。
- 高风险操作完成后强制拉取最新详情，避免前端脏状态。

## 6. 关键页面设计

### 6.1 认证页面
- 登录与注册表单采用统一校验规则与错误提示模型。
- 登录成功后持久化会话并刷新用户上下文。
- 注册成功后引导进入登录流程，失败时展示明确原因。

### 6.2 账户设置页面
- 交易账户列表支持新增、删除、刷新一致性。
- 模型设置区 MVP 展示与引导，V1 开放保存与生效反馈。
- 能力未开放时禁用写操作并显示版本提示。

### 6.3 场内策略页面
- 支持基础策略创建、参数校验、账户绑定。
- 支持启动、停止、更新、下线，关键操作需确认。
- 策略状态展示采用统一状态语义，删除显示为业务下线。

### 6.4 场外策略页面
- MVP 可先占位展示，V1 开放创建编辑与生命周期操作。
- 后端未交付时显示能力未开放并保留返回路径。
- 支持失败原因展示，避免静默失败。

### 6.5 已完成策略页面
- 展示完成原因、收益、创建时间、类型信息。
- 支持收益正负差异化表达与时间排序。
- 配置缺失时启用默认展示方案。

### 6.6 智能分析页面
- 支持多轮对话、消息状态、Markdown 展示。
- 支持上下文类型选择与上下文对象注入。
- 支持模型列表读取、模型设置读取与保存反馈。

### 6.7 券商登录辅助页面
- 支持账号输入、验证码输入与获取流程。
- 支持 OCR 能力占位与降级引导。
- 登录成功后回跳触发页面并恢复业务上下文。

### 6.8 文档中心页面
- 首次进入默认打开首篇文档。
- 目录切换触发正文加载并保留滚动定位策略。
- 支持加载、失败、空数据三类状态提示。

## 7. 接口契约与错误处理

### 7.1 统一响应结构
- status：success 或 error。
- message：用户可读结果描述。
- data：业务数据，失败时允许 null。
- error_code：成功为 null，失败为可检索错误码。
- request_id：单次请求追踪标识。
- timestamp：结果时间戳。

### 7.2 错误码分层
- AUTH-*：认证与会话错误。
- PERM-*：权限与主体一致性错误。
- BIZ-*：业务规则错误。
- EXT-*：外部依赖错误。
- SYS-*：系统异常错误。

### 7.3 页面反馈策略
- 关键动作统一 toast 或内联反馈，不使用不可读错误码直出。
- 可恢复错误提供重试入口，不可恢复错误提供返回路径。
- 高风险失败提示包含下一步操作建议。

## 8. 非功能设计

### 8.1 性能
- 核心页面首屏采用分段加载，先框架后数据。
- 列表页优先骨架屏，降低感知等待时间。
- 配置与低频数据启用缓存，缓存命中目标不低于 70%。

### 8.2 可用性
- 按钮文案、提示语、状态文案来自统一配置字典。
- 关键链路均有加载态、成功态、失败态。
- 移动端核心链路可在单手操作区完成。

### 8.3 可访问性
- 导航和主内容区具备清晰语义结构。
- 关键控件支持键盘聚焦和可见焦点样式。
- 反馈信息需文本可感知，不依赖纯颜色表达。

### 8.4 可维护性
- 业务域隔离，跨域调用通过服务层完成。
- 公共组件和表单能力复用统一基础库。
- 新增策略类型不修改既有页面消费契约。

## 9. 验收与测试设计

### 9.1 测试分层
- 单元测试：服务层、状态管理、工具函数。
- 组件测试：关键组件的输入校验与状态渲染。
- 端到端测试：登录、账户、策略、文档、智能分析核心链路。

### 9.2 MVP 验收重点
- FRT-001~FRT-008、FRT-013 必须全部可测并通过。
- 页面不得写死业务文案，抽样检查通过。
- 会话失效重登引导、权限拦截、错误反馈全链路生效。

### 9.3 V1 验收重点
- 场外策略、已完成策略、智能分析、券商辅助流程可用。
- 模型设置写入能力可保存并可读取回显。
- 未开放能力提示与版本口径与后端、CLI一致。
