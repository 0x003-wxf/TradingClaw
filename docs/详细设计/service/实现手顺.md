# TradingClaw 后端 Service 实现手顺

## 1. 目标

- 基于已产出的架构、边界、事件、状态机、API 契约与字段字典，给出可直接执行的开发顺序。
- 以高内聚低耦合为原则，保证每个模块能独立开发、独立验收、低风险合并。
- 开发人员按步骤推进即可，不依赖额外架构讨论。

## 2. 输入文档与使用顺序

先读文档顺序固定如下：

1. `服务架构方案.md`：明确总体边界与演进方向。
2. `服务边界设计.md`：明确模块职责和数据主权。
3. `状态机设计.md`：落实策略与计划状态迁移规则。
4. `事件模型设计.md`：落实异步链路、幂等、死信与回放。
5. `API契约草案.md`：落实接口清单与响应口径。
6. `API字段字典.md`：落实字段类型/必填/约束。

## 3. 实施总原则

- 单次只做一个“可闭环模块”，每个模块必须“接口+校验+存储+测试”同时完成。
- 模块间只通过契约交互，不跨模块直连数据表。
- 先做同步主链路，再补异步事件与扩展能力。
- 所有写接口默认幂等，所有错误统一错误码。

## 4. 模块落地顺序（必须按序）

M0 -> M1 -> M2 -> M3 -> M4 -> M5 -> M6 -> M7

- M0：公共底座（网关、鉴权、错误码、响应模型、幂等）
- M1：identity-service（USR）
- M2：market-service（MKT）
- M3：securities-trade-service + unified-trade-kernel（SEC+UTL）
- M4：strategy-service（STR）
- M5：planning-service（PLN，V1）
- M6：intelligence-service（AI/OCR，V1）
- M7：事件总线与观测增强（跨模块）

## 5. 按模块的“文件级”实现手顺

以下是每个模块建议的文件创建与编码顺序。每一步完成后立即跑本模块测试。

## 5.1 M0 公共底座

### 文件手顺

1. `src/common/contracts/response.ts`：统一响应模型。
2. `src/common/contracts/errors.ts`：错误码常量与映射。
3. `src/common/middleware/auth.ts`：token + 主体一致性校验。
4. `src/common/middleware/request_id.ts`：request_id 注入与透传。
5. `src/common/middleware/idempotency.ts`：幂等键校验与存储。
6. `src/common/http/router.ts`：统一路由挂载入口。

### 完成标准

- 任意接口都返回统一响应结构。
- 未携带鉴权头或幂等键时可返回标准错误码。
- 日志中可见 request_id 并全链路透传。

## 6. 联调手顺（前后端）

按接口分组依次联调，前一组通过后再进入下一组：

1. 认证组：register/login/profile
2. 行情组：realtime/history/ma
3. 交易组：session login -> order -> cancel -> orders/trades query
4. 策略组：create -> start/stop/offline/archive
5. 计划组：create -> publish-check -> publish -> validations
6. 智能组：chat -> model config -> ocr

联调统一检查：

- 字段名、类型、必填、约束严格对齐 `API字段字典.md`
- 错误码与 message 对齐 `API契约草案.md`
- 写接口必须携带 `Idempotency-Key`

## 7. 每个 PR 的提交粒度

- 规则一：一个 PR 只做一个模块或一个可闭环子能力。
- 规则二：每个 PR 必须同时包含接口实现与对应测试。
- 规则三：禁止“跨模块大杂烩”提交，避免回归风险扩散。

推荐 PR 顺序：

1. PR-01 公共底座
2. PR-02 identity-service
3. PR-03 market-service
4. PR-04 trade-kernel + securities-service
5. PR-05 strategy-service
6. PR-06 planning-service
7. PR-07 intelligence-service
8. PR-08 events + observability

## 8. 开发完成判定（DoD）

模块完成必须全部满足：

- 接口：契约与字段字典 100% 对齐。
- 规则：状态机、幂等、权限、归属校验全部生效。
- 测试：单元测试 + 集成测试通过。
- 质量：lint、typecheck、测试全通过。
- 可观测：日志、指标、追踪可用。
