# TradingClaw 行情与资讯详细设计

## 1. 范围说明

- 本文档覆盖 `market-service` 与 `news-service`。
- 对应需求主要包括 `MKT-001` ~ `MKT-004`、`NEWS-001`、`NEWS-002`。
- 目标是为交易、策略、AI 提供标准化的市场数据和资讯供给能力。

## 2. 模块职责

### 2.1 `market-service`

- 股票、基金、数字资产实时行情查询。
- 历史行情、K 线、均线等指标供给。
- 行情标准化、缓存回退、缺口识别与增量补齐。

### 2.2 `news-service`

- 多源资讯抓取、聚合、去重、分页查询。
- 资讯影响力评分与排序。

## 5. 核心业务流程

### 5.1 实时行情查询

1. 接收行情查询请求。
2. 优先从缓存读取热点数据。
3. 未命中时访问外部行情源。
4. 标准化后写回缓存并返回。
5. 外部依赖失败时回退可用缓存并标记数据状态。

### 5.2 历史行情补齐

1. 接收区间查询或离线回填任务。
2. 检查已有区间和缺口。
3. 通过 `HistoricalBackfillWorkflow` 补齐缺口。
4. 完成去重、排序、标准化后写入时序库。
5. 发布 `historical.backfilled` 事件。

### 5.3 指标计算

1. 读取标准化 K 线序列。
2. 根据指标类型和窗口进行计算。
3. 样本不足时返回不可计算状态。
4. 计算结果进入分析库并可发布 `indicator.calculated` 事件。

### 5.4 资讯聚合与评分

1. 从多个资讯源抓取原始新闻。
2. 标准化、清洗并建立去重指纹。
3. 入库标准资讯实体。
4. 执行影响力评分。
5. 写入检索索引并发布 `news.ingested`、`news.scored` 事件。

## 6. 数据设计

- 行情明细和 K 线不堆在事务库。
- 资讯正文可在事务库保留结构化摘要，在搜索引擎保留检索视图。

## 9. 依赖与实施顺序

- 本模块建议第二批实现。
- 是策略系统、AI、交易查询的重要基础数据依赖。
- 行情标准化口径必须在本模块统一，不允许策略和交易模块各自做二次定义。
