# TradingClaw 后端 Service 详细设计：状态机

## 1. 设计目标

- 统一策略与交易计划状态语义，避免跨端口径分裂。
- 确保状态迁移可校验、可追踪、可回放。
- 将关键状态变更纳入审计闭环，满足发布与复盘要求。

## 2. 通用状态机约束

- 约束一：状态变更必须先记录状态日志，再提交状态更新。
- 约束二：非法迁移必须阻断并返回标准错误码与可读原因。
- 约束三：每次迁移必须携带操作者、请求ID、时间戳、来源。
- 约束四：状态变更后必须发布领域事件供下游消费。
- 约束五：幂等请求不得导致重复迁移。

## 3. 策略状态机（STR）

## 3.1 状态集合

- pending_start：待启动
- running：运行中
- stopped：已停止
- offline：已下线
- archived：已归档

## 3.2 允许迁移

| 当前状态 | 触发动作 | 目标状态 | 前置校验 |
| --- | --- | --- | --- |
| pending_start | start | running | 会话有效、额度充足、账户归属通过 |
| running | stop | stopped | 操作主体一致、策略可停止 |
| stopped | restart | running | 会话有效、策略可恢复 |
| pending_start/running/stopped | delete | offline | 主体一致、未处于归档 |
| offline | archive | archived | 仅已下线可归档 |

## 3.3 禁止迁移

- archived -> running
- offline -> running
- archived -> pending_start
- 任何状态 -> 不存在状态

## 3.4 执行流程

- 第一步：校验会话、令牌、主体一致性、额度与账户归属。
- 第二步：校验状态迁移合法性。
- 第三步：写入状态日志表。
- 第四步：更新策略主表状态与版本号。
- 第五步：发布 strategy.state.changed 事件。

## 4. 交易计划状态机（PLN）

## 4.1 状态集合

- draft：草稿
- in_review：评审中
- approved：已批准
- published：已发布
- archived：已归档

## 4.2 允许迁移

| 当前状态 | 触发动作 | 目标状态 | 前置校验 |
| --- | --- | --- | --- |
| draft | submit_review | in_review | 计划身份唯一、必填字段完整 |
| in_review | approve | approved | 风险评审有效 |
| approved | publish | published | 发布检查通过、校验命中当前内容版本 |
| published | create_next_version | draft | 基于当前版本创建下一版本 |
| draft/in_review/approved/published | archive | archived | 非执行中且满足归档条件 |

## 4.3 发布检查规则

- 最新评审结论必须为可发布状态。
- 最新校验结果必须通过且绑定当前内容更新时间。
- 存在阻塞项时必须阻断并返回阻塞原因列表。

## 4.4 执行流程

- 第一步：读取计划当前状态与版本上下文。
- 第二步：执行迁移校验与发布可用性检查。
- 第三步：写入状态日志与迁移审计记录。
- 第四步：更新计划状态与版本指针。
- 第五步：发布 plan.* 领域事件。

## 5. 状态日志与审计模型

## 5.1 状态日志字段

| 字段 | 说明 |
| --- | --- |
| log_id | 日志唯一ID |
| domain_type | 领域类型（strategy/plan） |
| aggregate_id | 聚合根ID |
| from_state | 迁移前状态 |
| to_state | 迁移后状态 |
| action | 触发动作 |
| operator_id | 操作者 |
| trace_id | 请求ID |
| reason | 迁移说明或失败原因 |
| created_at | 创建时间 |

## 5.2 审计要求

- 所有成功迁移必须有日志。
- 所有失败迁移必须保留失败审计。
- 日志支持按用户、策略、计划、时间窗口检索。

## 6. 错误码建议

| 错误码 | 场景 | 说明 |
| --- | --- | --- |
| STATE_INVALID_TRANSITION | 非法状态迁移 | 当前状态不允许执行该动作 |
| STATE_PRECONDITION_FAILED | 前置条件不满足 | 会话、额度、归属、评审或校验不满足 |
| STATE_IDEMPOTENT_REPLAY | 幂等重复请求 | 请求已处理，不重复迁移 |
| STATE_VERSION_CONFLICT | 并发版本冲突 | 状态版本已变更，需重试 |

## 7. 并发与幂等策略

- 并发控制：状态主表采用乐观锁版本号更新。
- 幂等控制：以业务幂等键（aggregate_id + action + request_id）去重。
- 重试策略：仅对版本冲突和短暂依赖错误可重试。

## 8. 监控指标

- 状态迁移成功率
- 非法迁移拦截次数
- 状态迁移平均耗时
- 幂等命中率
- 发布检查阻断率

## 9. 验收检查项

- 策略状态机符合“待启动/运行中/已停止/已下线/已归档”口径。
- 计划状态机符合“草稿/评审中/已批准/已发布/已归档”口径。
- 迁移前置条件检查完整并可返回可读错误。
- 状态日志与领域事件可建立一一追踪关系。
